#### 크레인 인형뽑기 게임

- 알고리즘: (2차원) 배열의 인덱스, stack의 기본 개념

- 풀이과정

```
	풀이 시간 중 배열의 인덱스를 생각하는데 대부분의 시간이 소요됐다. 예시의 기준으로 board 배열을 받으면 (0, 0)부터 (4, 4) 인덱스까지 받게 된다. 이를 예시의 그림과 비교하면, 첫번째 행이 그림의 상단에 오게 되고 마지막 인덱스가 하단에 오게 된다.
	
	그림과 같은 형태를 파이썬 2차원 배열로 이해할 때는 보통 y축 방향 기준으로 아래 대칭시킨 형태로 생각해왔다. 이 문제도 그럴 것이라고 생각해서 노트에 입력 예시를 적고 보니 그림과 달라서 당황했다. 다시 보니 2차원 배열의 입력 모양과 그림의 모양이 같았다.
배열과 그림의 모양까지는 이해를 했으니 인덱스만 어떻게 할지 정하면 거의 정답에 접근하게 된다. 위에 쌓인 인형부터 뽑기가 진행되기 때문에 반복문으로 인덱스를 돌 때 입력받은데로 이중 for문을 돌면 된다.

	moves의 배열을 바깥 for문, board의 길이를 안쪽 for문으로 이중 for문을 돌렸다. board의 길이 값은 행값으로 하고 moves의 값은 열 값으로 한 board의 값이 0이 아니라면 스택에 넣었다. 단 moves의 1은 board의 0열과 같기 때문에 moves의 값은 1씩 뺐다.
스택에 막 들어온 값과 기존에 마지막이었던 값이 같으면 pop을 하고 pop한 인형의 숫자를 정답을 구할 변수에 더해서 답을 구했다.

p.s-프로그래머스 사이트에서 문제를 직접 푼 경험이 거의 없어서 입력을 받는 것에서 처음에 헤맸다. 알고보니 입력을 따로 받을 필요가 없었다.
```
